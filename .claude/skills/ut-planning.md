# 사용성 테스트(UT) 계획

## 목적
실제 사용자가 제품을 어떻게 사용하는지 관찰하여 UX 문제를 발견하고,
출시 전 개선 기회를 식별한다.

## UT 종류 선택 가이드

| 방법 | 적합한 상황 | 특징 |
|------|-----------|------|
| **모더레이트 UT** | 복잡한 플로우 탐색, 초기 프로토타입 | 실시간 질문, 깊은 인사이트 |
| **언모더레이트 UT** | 빠른 정량 데이터, 대규모 샘플 | 자동화, 낮은 비용 |
| **원격 UT** | 시간/지역 제약 | Zoom, Lookback 활용 |
| **대면 UT** | 섬세한 관찰 필요 시 | 얼굴 표정, 제스처 관찰 가능 |

## 단계별 수행 방법

### 1단계: UT 목적 및 가설 정의
- **테스트 목적**: 이 UT로 무엇을 알고 싶은가?
- **검증 가설**: "우리는 [사용자]가 [태스크]를 [방식]으로 수행할 것이라고 가설을 세운다"
- **핵심 질문 3개**: 이 UT 후 반드시 답해야 할 질문

### 2단계: 참여자 프로필 설정
- **모집 기준**: 타겟 사용자 프로필
- **모집 방법**: 내부 DB, 소셜미디어, 리쿠르팅 서비스
- **참여자 수**: 5-8명 (질적 UT 기준, Nielsen의 법칙)
- **인센티브**: 기프티콘, 크레딧, 현금 등
- **스크리너 질문**: 적합한 참여자 선별 기준

### 3단계: 테스트 시나리오 및 태스크 설계

**시나리오 예시:**
"당신은 팀 리더입니다. 이번 분기 팀 OKR을 처음으로 설정해야 합니다. 이 앱을 사용하여 OKR을 입력하고 팀원들과 공유해보세요."

**태스크 설계 원칙:**
- 과업 중심 (시나리오 기반) — "~를 찾으세요" (X) → "~를 완료하세요" (O)
- 시스템 지식을 가정하지 않음 ("메뉴를 클릭하세요" 금지)
- 5분 내 완료 가능한 크기로 설계

**태스크 난이도:**
- 쉬운 태스크 → 어려운 태스크 순서로 배치
- 태스크당 예상 소요 시간 명시

### 4단계: 측정 지표 정의

**정량 지표:**
| 지표 | 설명 | 측정 방법 |
|------|------|----------|
| 태스크 성공률 | 태스크를 완료한 참여자 비율 | 성공/실패 기록 |
| 완료 시간 | 태스크 완료에 걸린 시간 | 타이머 |
| 오류 횟수 | 잘못된 클릭/입력 횟수 | 관찰 기록 |
| SUS 점수 | System Usability Scale (0-100) | 10문항 설문 |

**SUS 10문항** (5점 척도):
1. 이 시스템을 자주 사용하고 싶다
2. 이 시스템은 불필요하게 복잡하다 (역채점)
3. 이 시스템은 사용하기 쉽다
... (나머지 문항 포함)

**SUS 점수 해석:**
- 85 이상: 우수
- 68-85: 양호 (업계 평균 68)
- 51-68: 보통 (개선 필요)
- 51 미만: 불량 (즉각 개선 필요)

**정성 지표:**
- Think-Aloud (소리 내어 생각하기) 발언 기록
- 감정적 반응 (좌절, 혼란, 놀라움)
- 예상치 못한 사용 패턴

### 5단계: UT 진행 가이드

**진행자(Moderator) 스크립트:**
```
[오프닝]
"안녕하세요. 오늘 시간 내주셔서 감사합니다.
저는 제품을 테스트하는 게 아니라 제품이 잘 작동하는지를 테스트하는 것입니다.
틀린 답변은 없으니 편하게 해주세요. 가능하면 생각하시는 것을 소리 내어 말씀해 주세요."

[태스크 진행 중]
- 힌트 제공 금지
- "지금 무슨 생각을 하고 계세요?" (중립적 프로브)
- "어떤 걸 찾고 계신가요?"

[클로징]
"마지막으로 이 제품에 대해 전반적인 느낌을 말씀해 주실 수 있나요?"
```

### 6단계: 결과 분석 및 보고

**레인보우 스프레드시트:**
- 행: 태스크 / 관찰 항목
- 열: 참여자별
- 셀: 성공(✅)/실패(❌)/부분 성공(⚠️)

**우선순위 결정:**
- 빈도 × 심각도 매트릭스
- 3명 이상 동일 문제 → Critical Issue

## 산출물 형식

```
## 사용성 테스트 계획서

### 테스트 목적
[UT로 알고 싶은 것과 검증 가설]

### 참여자
- 프로필: [기준]
- 모집 목표: [명]
- 인센티브: [내용]

### 테스트 태스크
1. 시나리오: [내용]
   태스크: [구체적 과업] (예상 소요: [분])
2. ...

### 측정 지표
- 정량: 성공률, 완료 시간, SUS 점수
- 정성: Think-Aloud 발언, 감정 반응

### 일정
- 모집: [기간]
- 진행: [날짜 및 시간]
- 분석 완료: [날짜]
- 결과 공유: [날짜]
```
