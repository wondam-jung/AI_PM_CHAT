# 사용자 인터뷰 설계

## 목적
고객의 진짜 문제와 니즈를 발견하고, 가설을 검증하기 위한 정성적 데이터를 수집한다.

## 인터뷰 설계 원칙
- 해결책이 아닌 문제를 탐색한다 (솔루션 피칭 금지)
- 과거 행동에 대해 묻는다 (미래 의도가 아닌 실제 경험)
- 열린 질문으로 대화를 이끈다 (Yes/No 질문 지양)
- 5Why 기법으로 근본 원인을 탐색한다

## 단계별 수행 방법

### 1단계: 인터뷰 목적 정의
- 검증하려는 핵심 가설 2-3개 명시
- 인터뷰 후 결정할 수 있어야 하는 것 정의
- 필요한 인터뷰 횟수: 5-8명 (포화점 도달 기준)

### 2단계: 참여자 모집 기준 설정
- 타겟 페르소나 프로필 정의 (역할, 규모, 경험 수준)
- 포함 기준 및 제외 기준
- 모집 채널 (LinkedIn, 커뮤니티, 고객 DB 등)
- 인센티브 설계 (기프티콘, 제품 크레딧 등)

### 3단계: 질문지 설계

**오프닝 (5분)**
- 인터뷰 목적 및 방식 안내
- 동의 확인 (녹음 포함)
- 아이스브레이커: "간단히 자기소개 부탁드려요"

**배경 파악 (10분)**
- "현재 [업무/상황]을 어떻게 처리하고 계신가요?"
- "가장 최근에 [관련 상황]을 경험하셨을 때를 얘기해 주실 수 있나요?"
- "그 상황에서 가장 어려웠던 부분은 무엇이었나요?"

**문제 탐색 (15분)**
- "그 문제가 얼마나 자주 발생하나요?"
- "그 문제 때문에 어떤 영향을 받으셨나요?"
- "지금은 어떻게 해결하고 계신가요?" (현재 대안 파악)
- "그 방법에서 아쉬운 점은 무엇인가요?"

**5Why 프로브 질문**
- "왜 그렇게 생각하세요?"
- "조금 더 자세히 얘기해 주실 수 있나요?"
- "그때 어떤 감정이 드셨나요?"

**클로징 (5분)**
- "오늘 인터뷰에서 다루지 못한 중요한 내용이 있을까요?"
- "혹시 소개해 주실 수 있는 비슷한 상황의 분이 계신가요?"
- 감사 인사 및 다음 단계 안내

### 4단계: 인사이트 분석 템플릿

인터뷰별 요약:
- 참여자: [이름/역할]
- 핵심 문제: [발견된 핵심 문제 1-3개]
- 현재 대안: [현재 사용 중인 해결 방법]
- 인상적인 발언: [그대로 인용]
- 검증된 가설: [O/X]
- 새로운 인사이트: [예상치 못한 발견]

### 5단계: 패턴 종합
- 5명 이상 공통으로 언급된 문제 → 핵심 문제 확정
- 2-3명만 언급한 문제 → 추가 검증 필요
- 1명만 언급한 문제 → 예외 케이스

## 산출물 형식

```
## 사용자 인터뷰 계획서

### 목적
[인터뷰로 검증할 가설과 결정 사항]

### 참여자 프로필
- 직군/역할: [예: B2B SaaS PM, 팀 규모 10인 이상]
- 경험: [예: 제품 관리 경력 2년 이상]
- 모집 목표: [5-8명]

### 질문지
[오프닝 → 배경 → 문제 탐색 → 클로징]

### 일정
- 모집 기간: [기간]
- 인터뷰 기간: [기간]
- 분석 완료: [날짜]
```
