# 기능 우선순위 검증

## 목적
제한된 리소스로 최대의 비즈니스 가치를 창출하기 위해 데이터와 프레임워크 기반으로
기능의 우선순위를 객관적으로 결정한다.

## 우선순위 프레임워크

### 1. RICE 스코어링
각 기능에 대해 4가지 요소를 평가하여 점수를 산출한다.

**공식: RICE Score = (Reach × Impact × Confidence) ÷ Effort**

| 요소 | 설명 | 측정 기준 |
|------|------|----------|
| **Reach** | 분기 동안 영향받는 사용자 수 | 실제 사용자 수 (예: 1,000명) |
| **Impact** | 개인 사용자에게 미치는 영향 | 3=대규모, 2=높음, 1=보통, 0.5=낮음, 0.25=최소 |
| **Confidence** | 추정치에 대한 확신 수준 | 100%=높음, 80%=보통, 50%=낮음 |
| **Effort** | 소요 시간 (인-월, person-months) | 0.5=2주, 1=1개월, 3=분기 |

**예시:**
```
기능 A: Reach=500, Impact=2, Confidence=80%, Effort=1개월
RICE = (500 × 2 × 0.8) ÷ 1 = 800

기능 B: Reach=1000, Impact=1, Confidence=50%, Effort=0.5개월
RICE = (1000 × 1 × 0.5) ÷ 0.5 = 1000
→ 기능 B 우선
```

### 2. MoSCoW 분석

| 분류 | 기준 | 비율 |
|------|------|------|
| **Must Have** | 없으면 제품 실패. 법적/안전 요구사항 포함 | ~60% |
| **Should Have** | 중요하지만 없어도 임시 방편 존재 | ~20% |
| **Could Have** | 있으면 좋지만 없어도 무방 | ~20% |
| **Won't Have** | 이번 릴리즈에서 의도적으로 제외 | 나머지 |

**분류 기준 질문:**
- 이 기능 없이도 제품이 출시될 수 있는가? → Yes → Should 이하
- 이 기능 없이는 주요 고객 흐름이 막히는가? → Yes → Must Have

### 3. Value vs Effort 매트릭스

```
높은 가치
    │    빠른 승리(Quick Wins) │   전략적 도전(Big Bets)
    │         [우선 실행]      │      [장기 계획 수립]
    ├─────────────────────────┼──────────────────────────
낮은 가치│    채우기(Fill-ins)  │      피해야 할 것(Avoid)
    │       [나중에 고려]      │        [제외 검토]
    └─────────────────────────────────────────────────
              낮은 노력              높은 노력
```

→ **Quick Wins 우선 집중**: 높은 가치 + 낮은 노력

## 단계별 수행 방법

### 1단계: 기능 목록 정리
평가할 기능들을 명확히 정의하고 목록화

### 2단계: RICE 평가
각 기능에 대해 Reach, Impact, Confidence, Effort 수치 입력 후 점수 산출

### 3단계: MoSCoW 분류
RICE 점수와 비즈니스 맥락을 고려하여 Must/Should/Could/Won't 분류

### 4단계: Value vs Effort 매핑
직관적 시각화로 Quick Wins 식별

### 5단계: 최종 우선순위 확정
- 이해관계자 리뷰 및 합의
- 스프린트/릴리즈 계획에 반영
