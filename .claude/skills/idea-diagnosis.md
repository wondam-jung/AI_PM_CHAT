# 아이디어 진단 및 검증

## 목적
아이디어에 내재된 핵심 가정을 파악하고, 최소한의 비용으로 빠르게 검증한다.

## 단계별 수행 방법

### 1단계: 아이디어 명확화
아이디어를 한 문장으로 요약한다.
형식: "[타겟 고객]의 [문제]를 [솔루션]으로 해결하여 [가치]를 제공한다"

### 2단계: 핵심 가정 도출
아이디어가 사실이려면 참이어야 하는 전제들을 모두 나열한다.

| 가정 유형 | 질문 | 예시 |
|----------|------|------|
| 고객 가정 | 이 고객 그룹이 실제 존재하는가? | "PM들이 OKR 설정에 어려움을 느낀다" |
| 문제 가정 | 이 문제가 그들에게 실제로 중요한가? | "매주 30분 이상 OKR 관련 업무에 시간을 쓴다" |
| 솔루션 가정 | 우리 방식이 문제를 해결할 수 있는가? | "AI가 작성해준 OKR 초안이 유용할 것이다" |
| 비즈니스 가정 | 수익화할 수 있는가? | "월 3만원에 기꺼이 지불할 것이다" |

### 3단계: 가정 우선순위화 (2x2 매트릭스)
- X축: 불확실성 (낮음 → 높음)
- Y축: 중요도 (낮음 → 높음)

→ 우선 검증 대상: **불확실성 높음 + 중요도 높음**

### 4단계: 린 캔버스 작성

| 항목 | 내용 |
|------|------|
| 문제 | 고객이 겪는 상위 3가지 문제 |
| 고객 세그먼트 | 얼리어답터 프로필 |
| 고유한 가치 제안 | 한 줄 메시지 |
| 솔루션 | 상위 3가지 기능 |
| 채널 | 고객에게 도달하는 방법 |
| 수익원 | 수익 모델 |
| 비용 구조 | 주요 비용 항목 |
| 핵심 지표 | 성과를 측정하는 지표 |
| 경쟁 우위 | 쉽게 복사할 수 없는 요소 |

### 5단계: 검증 실험 설계
우선순위가 높은 가정 1-2개에 대한 검증 실험을 설계한다.

| 가정 | 검증 방법 | 성공 기준 | 기간 | 비용 |
|------|----------|----------|------|------|
| [가정 1] | [인터뷰/랜딩/프로토타입] | [정량 기준] | [기간] | [비용] |

**검증 방법 선택 기준:**
- 고객/문제 가정 → 사용자 인터뷰 (5-8명)
- 솔루션 가정 → Wizard of Oz, 프로토타입 테스트
- 시장 가정 → 랜딩 페이지, 광고 실험
- 비즈니스 가정 → 가격 실험, 사전 판매

## 산출물 형식

```
## 아이디어 진단 결과

### 아이디어 요약
[한 문장 요약]

### 핵심 가정 TOP 3 (우선 검증 필요)
1. [가장 위험한 가정]
2. [두 번째 가정]
3. [세 번째 가정]

### 린 캔버스
[표 형식]

### 검증 실험 계획
[가정 → 방법 → 기준 → 기간]

### 권고사항
- 즉시 시작: [첫 번째 실험]
- 전제 조건: [먼저 확인해야 할 것]
```
