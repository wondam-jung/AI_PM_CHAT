# OKR 정의 및 분석

## 목적
조직과 팀의 방향성을 정렬하고, 측정 가능한 목표를 통해 성과를 추적한다.

## OKR 작성 원칙

### Objective (목표) 기준
- 도전적이고 영감을 주는 질적 목표
- 3개월 또는 분기 단위로 설정
- 팀이 기억하고 공감할 수 있을 만큼 간결하게 작성
- 측정 방법이 아닌 방향성을 표현

### Key Results (핵심 결과) 기준
- 목표당 3-5개 설정
- 반드시 수치로 측정 가능해야 함 (%, 건수, 금액, 시간 등)
- 달성 여부가 명확히 판단 가능해야 함
- 50-70% 달성이 도전적인 수준이 적절 (100% 달성은 쉬운 목표)

## 단계별 수행 방법

### 1단계: 비즈니스 컨텍스트 파악
- 현재 분기/연도의 회사 최우선 과제
- 이전 OKR 달성률 및 교훈
- 팀/개인 OKR이 상위 OKR과 연결되는 방식

### 2단계: Objective 작성
형식: "[동사] + [명사구] + [이유/목적]"
예시: "고객이 열광하는 온보딩 경험을 만들어 초기 이탈을 줄인다"

좋은 Objective 체크리스트:
- [ ] 팀이 에너지를 받는가?
- [ ] 분기 내 달성 가능한가?
- [ ] 측정 가능한 Key Results와 연결되는가?

### 3단계: Key Results 작성
형식: "[지표]를 [현재값]에서 [목표값]으로 [기간]까지 달성한다"
예시: "신규 사용자 7일 리텐션율을 30%에서 50%로 Q2 내 달성한다"

나쁜 KR 패턴 (피할 것):
- 활동 기반: "온보딩 플로우 개선 작업을 완료한다" (→ 결과로 전환)
- 이진 완료형: "A/B 테스트를 한다" (→ 수치 결과로 전환)
- 모호한 수치: "개선한다", "향상시킨다"

### 4단계: OKR 정렬 검토
- 상위 OKR과의 연결 관계 확인
- 팀 간 OKR 충돌 또는 중복 식별
- 리소스 배분 균형 검토

### 5단계: OKR 리뷰 계획
- 주간: 진행 상황 체크인 (신호등: 🟢 정상/🟡 위험/🔴 차단)
- 월간: 중간 점검 및 재조정
- 분기 말: 달성률 평가 및 학습 정리

## 산출물 형식

```
## [팀명/제품명] OKR - [분기/연도]

### Objective 1: [목표 문장]
- KR 1: [지표]를 [현재값]에서 [목표값]으로 달성
- KR 2: [지표]를 [현재값]에서 [목표값]으로 달성
- KR 3: [지표]를 [현재값]에서 [목표값]으로 달성

### Objective 2: [목표 문장]
- KR 1: ...

### OKR 정렬
- 상위 OKR: [연결되는 회사/조직 OKR]
- 협력 팀: [함께 달성해야 할 팀]
```
